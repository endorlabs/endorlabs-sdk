# Optional User Rules for Cursor

Optional rules you can copy into **Cursor Settings → General → Rules for AI** so they apply across all your projects.

Examples of preferences you might add:

- **TDD**: Prefer test-driven development; write or update tests when changing behavior.
- **OS-agnostic scripts**: Prefer cross-platform commands (e.g. `uv run`); avoid shell-specific syntax when documenting.
- **Consistency**: Follow project conventions and existing patterns before adding new ones.

Customize this file for your own global AI rules, then copy the contents into Cursor’s Rules for AI.
